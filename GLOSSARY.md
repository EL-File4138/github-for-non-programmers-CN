# 版本控制

在编程中，版本控制是一种追踪代码随时间的变化的手段。这意味着任何开发过程中引入的错误都能够被轻松溯源并纠正。你也不需要担心你所使用的版本是不是最新版本——版本控制系统会追踪是否发生改变。

# Git

最广泛使用的版本控制系统之一。通常在命令行中使用，但是我们只会通过 GitHub Desktop 使用以保持教程简单。

# 仓库（Repository）

常缩写为“Repo”。仓库指存放所有希望被追踪的文件的项目文件夹。

# 提交（Commit）

仓库在某个时间点上的文件快照。你需要手动去执行提交操作，并且需要为每个提交附带一个描述改动的信息。

# GitHub

一个存放仓库的在线拷贝的网站，同时还允许其他人和你同时在同一个仓库中协作，并追踪每个人的改动以避免冲突。

# 事务（Issue）

GitHub的一个功能，允许为仓库创建一个待办列表。通过提交事务来指定需要解决的问题、提出疑问、建议新功能等。

# 分支（Branch）

分支是从某一个提交开始的仓库副本。你可以在一个分支中创建新工作而不需要担心影响其他分支。之后，你也可以将你的分支合并入其他分支，或者删除你的分支和其上的新工作（比如尝试的失败）。分支功能能让你在不危害现有成果的情况下加入新东西。

# 拉取请求（Pull Request）

GitHub的一个功能，允许你提出合并两个分支的请求。协作者们可以讨论合并的质量，而在拉取请求提交之后在分支上的新提交也会被加入拉取请求。一旦达成共识，分支可以被并入或者被拒绝。
